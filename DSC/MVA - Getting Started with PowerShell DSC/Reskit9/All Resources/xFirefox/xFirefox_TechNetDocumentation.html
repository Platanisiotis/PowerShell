Introduction

The xFirefox module is a part of the Windows PowerShell Desired State Configuration (DSC) Resource Kit, which is a collection of DSC Resources produced by the PowerShell Team. This module contains the MSFT_xFirefox resource. This DSC Resource allows you to install the Firefox

All of the resources in the DSC Resource Kit are provided AS IS, and are not supported through any Microsoft standard support program or service. The ""x" in xFirefox stands for experimental, which means that these resources will be fix forward and monitored by the module owner(s).

Please leave comments, feature requests, and bug reports in the Q & A tab for this module.

If you would like to modify xFirefox module, feel free. When modifying, please update the module name and resource name (instructions below). As specified in the license, you may copy or modify this resource as long as they are used on the Windows Platform.

For more information about Windows PowerShell Desired State Configuration, check out the blog posts on the PowerShell Blog (this is a good starting point). There are also great community resources, such as PowerShell.org, or PowerShell Magazine. For more information on the DSC Resource Kit, check out this blog post.

Installation

To install xFirefox module

  • Unzip the content under $env:ProgramFiles\WindowsPowerShell\Modules folder

To confirm installation:

  • Run Get-DSCResource to see that xFirefox is among the DSC Resources listed

Requirements

This module requires the latest version of PowerShell (v4.0, which ships in Windows 8.1 or Windows Server 2012R2). To easily use PowerShell 4.0 on older operating systems, install WMF 4.0. Please read the installation instructions that are present on both the download page and the release notes for WMF 4.0.

Description

The xFirefox module contains the MSFT_xFirefox DSC Resource. This DSC Resource allows you to install the Firefox of desired version.

Details

MSFT_xFirefox resource has following optionally properties:

  • Language: Specify the language of the browser to be installed.Default it is English.
  • VersionNumber: Specify the version number of the browser to be installed.Default it installs latest version.
  • OS: Specify the operating system on which the browser is to be installed. By default the operating system is windows.
  • MachineBits:Specifies the machine's operating system bit number. By default it is taken as x86.
  • LocalPath:The local location on the machine which can you used to place the downloaded installation file.

    • Renaming Requirements

    When making changes to these resources, we suggest the following practice:

    1. Update the following names by replacing MSFT with your company/community name and replacing the "x" with "c" (short for "Community") or another prefix of your choice:
      • Module name (ex: xFirefox becomes cFirefox)
      • Resource folder (ex: MSFT_xFirefox becomes Contoso_cFirefox)
      • Resource Name (ex: MSFT_xFirefox becomes Contoso_cFirefox)
    2. Update module and metadata information in the module manifest
    3. Update any configuration that use these resources

    We reserve resource and module names without prefixes ("x" or "c") for future use (e.g. "MSFT_Firefox" or "Firefox"). If the next version of Windows Server ships with a "Firefox" resource, we don't want to break any configurations that use any community modifications. Please keep a prefix such as "c" on all community modifications.
